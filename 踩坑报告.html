1️⃣ 项目背景

合同对比助手是一款专业的 Word/PDF 文档差异对比工具，专为法律工作场景设计。主要功能包括：

  • 自动识别文字增删改
  • 表格、页眉页脚、脚注、图片识别
  • 生成详细改动列表和 Excel 报告
  • 支持 Word 和 PDF 格式

2️⃣ 关键 Bug 与解决方案

Bug #1: 程序无法启动 - IndentationError

错误信息：
IndentationError: unexpected indent
File "core\doc_comparator.py", line 20

问题原因：

  • doc_comparator.py 第 20 行缺少 class DocumentComparator: 类定义
  • 文档字符串缩进错误，前面有多余空格

错误代码：

import hashlib
import io
    """文档对比器"""  ← 错误：缩进且缺少类定义
    
    def __init__(self):

正确代码：

import hashlib
import io


class DocumentComparator:  ← 添加类定义
    """文档对比器"""  ← 正确缩进
    
    def __init__(self):
教训：Python 对缩进极其敏感，打包后错误信息会模糊化。使用 IDE 的语法检查功能可以提前发现。

Bug #2: 页眉页脚提取失败 - AttributeError

错误信息：
AttributeError: '_Header' object has no attribute 'text'
File "core\doc_comparator.py", line 105

问题原因：

  • python-docx 库中，section.header 是对象，不是字符串
  • 需要遍历其中的 paragraphs 来获取文本内容

错误代码：

header_text = section.header.text.strip()  ← 错误

正确代码：

header_text = " ".join([p.text.strip() for p in section.header.paragraphs]).strip()
教训：python-docx 的 API 文档不够详细，需要实际测试。页眉页脚是复合对象，不是简单字符串。

Bug #3: 位置显示不直观

问题描述：详细改动列表中显示「位置：0」「位置：5」，用户需要自己计算是第几段。

修改前 修改后
位置：0 第 1 段
位置：5 第 6 段
位置：100 第 101 段
教训：程序员思维（从 0 开始）≠ 用户思维（从 1 开始）。位置信息应该直观，不需要用户计算。

3️⃣ 代码优化经验

优化 #1: 版本号动态读取

问题：版本号硬编码在多个文件中，每次更新需要修改多处。

解决方案：创建 get_version() 函数，从 version.json 统一读取。

def get_version():
    try:
        with open('version.json', 'r', encoding='utf-8') as f:
            data = json.load(f)
            return data.get('version', '1.0')
    except Exception:
        return '1.0'

VERSION = get_version()

4️⃣ 打包部署问题

问题 #1: macOS 无法打包 Windows exe

解决方案：PyInstaller 打包是平台相关的。必须在目标平台（Windows）上打包。

正确流程：

  1. macOS 开发、调试、修复代码
  2. git 同步代码到 Windows
  3. Windows 上运行 python build.py
  4. 生成 合同对比助手.exe

问题 #2: 打包后缺少依赖

解决方案：在 build.py 的 hiddenimports 中添加缺失模块。

问题 #3: 网络问题导致 git 推送失败

fatal: unable to access 'https://github.com/...': Error in the HTTP2 framing layer

解决方案：

  • 配置 git 使用 HTTP/1.1：git config --global http.version HTTP/1.1
  • 增加缓冲区大小：git config --global http.postBuffer 524288000
  • 使用 --force 强制推送
